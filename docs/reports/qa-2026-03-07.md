### Findings

#### Passed

- `backend/**` exposes the MVP scaffold routes recorded in `docs/adr/0008-predict-strike-mvp-scope-and-initial-stack.md`.
- `POST /api/v1/risk/score` returns an explainable weighted score with breakdown and thresholds, matching the transparent-scoring direction in `docs/specs/initial-draft-project-spec.md`.
- `GET /api/v1/markets/opportunities` returns model-vs-market comparison outputs consistent with the current scaffold scope.
- Spec-traceable acceptance tests were added for `docs/specs/initial-draft-project-spec.md` and `docs/api/openapi.yml`.

#### Gaps / Deviations

- `docs/api/openapi.yml` defines `SignalSource` with `additionalProperties: false`, fields `name`, `status`, `last_checked_at`, and allowed statuses `planned`, `active`, `degraded`, but the backend returns an extra `category` field and uses `stubbed` status instead. See `docs/api/openapi.yml`, `backend/src/models/schemas.py`, and `backend/src/services/seed_data.py`.
- Because of that drift, the OpenAPI-derived acceptance test in `tests/acceptance/test_openapi_yml_acceptance.py` fails against the running FastAPI app.
- The broader product-spec items for collectors, analyst alerts, and dashboard flows remain future work and were not validated as implemented features in this scaffold pass.
